# 정산 유형 분류 레퍼런스

## 분류 기준

| 유형 | 정산 금액 계산 | 정산 방식 | 협의 필요 팀 |
|------|-------------|---------|-----------|
| 유형1 | 판매 수수료 정산 (판매가 x 수수료율) | 예약 후 정산 (후정산) | 매출관리팀 |
| 유형2 | 공급가 정산 (공급가 x 판매수량) | 밸런스 충전 (선정산) | 매출관리팀, 자금팀 |
| 유형3 | 공급가 정산 (공급가 x 판매수량) | 예약 후 정산 (후정산) | 매출관리팀 |
| 유형4 | 기타 | 기타 | 개발팀, 매출관리팀, 자금팀 |

## 분류 로직

**Q1: 공급사가 제공하는 가격 구조는?**
- 공급사가 판매가를 정하고, 마리트가 수수료를 받는 구조 → **판매 수수료 정산**
  - 입점 파트너와 동일하게 판매 수수료율을 정산하는 형태
  - 공급사에서 정해진 판매가를 마리트에서도 그대로 사용
  - 수수료율에 따라 마리트 수익/파트너 수익 분배
- 공급사가 공급가(net_price / supply_price)를 제공하고, 마리트가 판매가를 직접 설정 → **공급가 정산**
  - API로 제공되는 공급가를 파트너에게 정산
  - 판매가는 공급가 기반으로 마리트가 직접 제어

**Q2: 정산 시점은?**
- 미리 밸런스/크레딧을 충전해두고 예약 시 차감 → **밸런스 충전 (선정산)**
- 예약 후 일정 주기로 정산 → **예약 후 정산 (후정산)**

## 유형별 체크리스트

### 유형1: 판매 수수료 + 후정산

- [ ] 매니저 페이지에서 수수료 정책 등록 (파트너ID 단위 필수)
  - 매니저 페이지: https://manager.myrealtrip.com/settle/partner-commission
  - 상품 및 옵션 단위는 자유롭게 등록 가능
- [ ] 파트너사에 정산 금액 확인 방법 별도 안내 (26년 1월 기준 파트너 페이지에서 정산 금액만 확인 가능)
- [ ] 매출관리팀과 협의

### 유형2: 공급가 + 선정산 (밸런스 충전)

- [ ] 판매가 정책 설정 (공급가 기반 마진 정책)
  - 매니저 페이지: https://manager.myrealtrip.com/crm/experiences-sale-price-policies
- [ ] MSP(최저판매가)가 정해진 상품 확인 및 관리
- [ ] **밸런스 관리 체계 수립 (매우 중요)**
  - 밸런스 0 → 예약 생성 불가
  - 성수기/프로모션 시 급격한 차감 대비 필요
  - 특정 금액 이하 시 자동 알림/충전 프로세스 구축
- [ ] 정산 주기 & 탑업 방식 결정
  - **내부 정산**: 마리트가 정해둔 주기대로 알아서 대사 및 송금 (운영 리소스 최소)
  - **별도 정산**: 공급사 주기에 맞춰 인보이스 상호 대사 후 송금
- [ ] 밸런스 금액 산정 (예상 매출액 + 버퍼)
- [ ] **계약서 확정 전에 매출관리팀/자금팀과 반드시 협의!**
  - 계약 완료 후 정산 방식이 안 맞으면 수기 운영 리소스가 지속 발생
- [ ] 매출관리팀 + 자금팀과 협의

### 유형3: 공급가 + 후정산

- [ ] 판매가 정책 설정 (공급가 기반 마진 정책)
- [ ] MSP가 정해진 상품 확인 및 관리
- [ ] **시차 주의**
  - 한국 기준 vs 공급사 현지 시간 기준 차이로 예약건수/금액 불일치 가능
  - 예: 한국 기준 1월 31일까지 100건 → 파리 기준 97건 (3건 차이 → 다음 달로 이월)
  - 정산 기준 시간 (마리트 기준 vs 공급사 기준) 명확히 합의 필요
- [ ] 인보이스 기준 정산 vs 마리트 내부 기준 정산 결정
  - 공급사 인보이스 기준 → 공급사 시간에 맞춤
  - 마리트가 인보이스 발행 또는 내부정산 → 마리트 기준
- [ ] 약속된 주기에 맞춰 정산 금액 송금/관리 (신뢰 유지)
- [ ] 매출관리팀과 협의

### 유형4: 기타

- [ ] 위 유형에 해당하지 않는 특수 케이스
- [ ] 개발팀 + 매출관리팀 + 자금팀과 함께 논의 필요
- [ ] 미확인 항목 정리 후 재분류
