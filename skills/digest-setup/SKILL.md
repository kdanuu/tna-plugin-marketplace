# Daily Digest 설정 관리

**IMPORTANT**: 모든 사용자 커뮤니케이션은 한국어로 한다.

## 실행 흐름

### 1. 현재 설정 확인

Read 도구로 `~/.claude/daily-digest.json` 읽기.

- 파일 없음 → "아직 설정이 없습니다. `/daily-digest`를 실행하면 온보딩이 시작됩니다." 출력 후 종료
- 파일 있음 → 현재 설정 요약 출력:

```
📋 현재 설정
━━━━━━━━━━━━
회의록 소스:
  ✅ caret (서버: caret)
  ❌ Google Meet (비활성)

전송 채널:
  ✅ 슬랙 AgentDeck 봇 (User ID: U12345678)
  ✅ poke
```

### 2. `--reset` 플래그 처리

`--reset`이 지정된 경우:

```
⚠️ 모든 설정을 초기화합니다. 정말 초기화하시겠습니까?
```

- Y → `~/.claude/daily-digest.json` 삭제 → "설정이 초기화되었습니다. `/daily-digest`로 다시 온보딩을 시작하세요." 출력
- N → 종료

### 3. 변경 메뉴 (--reset 아닐 때)

```
무엇을 변경하시겠어요?
[1] 회의록 소스 변경 (추가/제거)
[2] 전송 채널 변경 (추가/제거)
[3] 전체 초기화
```

#### [1] 소스 변경

현재 활성화된 소스를 보여주고:
- 비활성 소스 → "활성화하시겠어요?" → 활성화 시 연결 테스트 진행 (SKILL.md Phase 1과 동일)
- 활성 소스 → "비활성화하시겠어요?" → 단, 최소 1개는 유지해야 함

#### [2] 채널 변경

현재 활성화된 채널을 보여주고:
- 비활성 채널 → "활성화하시겠어요?" → 활성화 시 연결 테스트 진행 (SKILL.md Phase 2와 동일)
- 활성 채널 → "비활성화하시겠어요?" → 단, 최소 1개는 유지해야 함

#### [3] 전체 초기화

`--reset`과 동일한 플로우.

### 4. 설정 저장

변경 사항을 `~/.claude/daily-digest.json`에 원자적으로 저장.
Bash로 `chmod 0600 ~/.claude/daily-digest.json` 실행.

```
✅ 설정이 업데이트되었습니다.
{변경된 항목 요약}
```
