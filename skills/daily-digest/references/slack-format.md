# 슬랙 DM 리포트 포맷 가이드

## 전송 구조

제목은 DM으로 직접 전송하고, 상세 내용은 스레드로 전송한다.

### 제목 메시지 (DM 본문)

```
📋 오늘의 회의 리포트 (`{YYYY-MM-DD}`) — `{N}`건
```

- 1줄로 작성
- 강조는 backtick(`` ` ``)만 사용. `*bold*`, `_italic_` 사용 금지

### 스레드 본문

```
> `🗓 회의 {N}건`

• `[{회의 제목}]` {시작시간}-{종료시간} ({소스: caret/Gemini})
  참석: {참석자 목록}
  요약: {2-3문장 핵심 요약}
  액션 아이템:
  • @{담당자} — {할 일} ({마감일})
  • @{담당자} — {할 일} ({마감일})

• `[{회의 제목}]` {시작시간}-{종료시간} ({소스})
  ...

> `💡 오늘의 핵심`

• 총 액션 아이템: `{N}`건 (내 담당: `{M}`건)
• 다음 회의: {다음 회의 정보}
```

## 작성 규칙

- **Slack mrkdwn 문법만 사용**: `*bold*`, `_italic_` 사용 금지. 강조는 backtick(`` ` ``) 코드 포맷만 사용
- `_`가 포함된 이름은 반드시 backtick으로 감싸서 Slack이 italic으로 변환하는 것을 방지
- 섹션 제목: `> ` (blockquote) + backtick으로 키워드 강조
- 항목: `• ` 1단계만 사용. 중첩 불릿(`◦`) 사용 금지 (Slack은 들여쓰기를 무시함)
- 회의는 시간순으로 정렬
- 소스 표시: 각 회의 옆에 `(caret)` 또는 `(Gemini)` 표시
- 참석자: 이름만 나열 (이메일 제외)
- 요약: 핵심 결정사항 위주, 2-3문장 이내
- 액션 아이템: 담당자 + 할 일 + 마감일 형식
- "오늘의 핵심" 섹션: 내 담당 액션 아이템 수와 다음 일정 강조
- 본문 5,000자 초과 시 여러 스레드 답글로 분할
- 회의 0건일 때는 이 포맷을 사용하지 않음 (스킬에서 조기 종료)
