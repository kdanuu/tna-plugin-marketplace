# 슬랙 AgentDeck 봇 리포트 포맷 가이드

## 전송 구조

**전송 순서가 중요하다.** 내 할 일 → 회의별 정리본 → 종합.

### 1. 첫 번째: 내 할 일 (별도 DM, 가장 먼저 전송)

```
🚨 오늘 내 할 일 (`{N}`건)
• {할 일 1} — {마감일}
• {할 일 2} — {마감일}
• {할 일 3} — {마감일}
```

- 모든 회의에서 나온 **내 담당 액션 아이템만** 추출하여 가장 먼저 전송
- 마감일 임박(3일 이내)한 건은 `🔴` 표시: `🔴 {할 일} — 내일까지`
- 할 일이 0건이면: `✅ 오늘 새로운 할 일 없음`

### 2. 회의별 정리본 (각각 별도 DM)

#### DM 제목 (본문)

```
📋 {핵심 결정 또는 임팩트 한 줄} — `[{회의 제목}]`
```

- **회의 제목이 아니라 핵심 결정/임팩트가 먼저 온다**
- 예시:
  - `📋 Google Ads 예산 2배 증액 확정 — [해외렌트카] 마케팅 플랜 논의`
  - `📋 2분기 OKR 3개 확정, AI 네이티브 전환 착수 — [T&A] 2분기 플래닝`
  - `📋 결정 사항 없음, 다음 주 재논의 — [백엔드] 아키텍처 리뷰`
- 회의 1건당 DM 1개, 시간순으로 전송
- 강조는 backtick(`` ` ``)만 사용. `*bold*`, `_italic_` 사용 금지

#### 스레드 본문

```
`{시작시간}-{종료시간}` ({소스: caret/Gemini})
참석: {참석자 목록}

> `📌 배경`

{회의가 열린 이유, 이전 회의에서 이어진 맥락 등 1-2문장}

> `💬 논의 내용`

• {논의 주제 1}: {어떤 의견이 나왔고 어떤 결론이 났는지}
• {논의 주제 2}: {찬반 논의가 있었다면 양쪽 요약 포함}
• {논의 주제 3}: {미결 사항이 있으면 명시}

> `✅ 결정 사항`

• {확정된 결정 1}
• {확정된 결정 2}

> `📝 액션 아이템`

• @{담당자} — {할 일} ({마감일})
• @{담당자} — {할 일} ({마감일})
```

### 3. 마지막: 오늘의 종합 (별도 DM)

```
💡 오늘의 회의 종합 (`{YYYY-MM-DD}`) — `{N}`건
• 주요 결정: {핵심 결정 1-2개}
• 다음 회의: {다음 회의 정보}
```

## 작성 규칙

- **DM 제목은 핵심 임팩트**: 회의 이름이 아닌, "그래서 뭐가 결정됐는데?"를 한 줄로. 결정 사항이 없으면 "결정 사항 없음, {다음 단계}" 명시
- **정리본 스타일**: 요약이 아닌 정리본. 회의의 맥락, 논의 과정, 결정 근거를 충분히 담는다
- 논의 내용은 주제별로 정리하되, 어떤 의견이 오갔는지 흐름이 보이도록 작성
- 결론 없이 끝난 논의는 "미결" 표시하고 다음 단계를 명시
- 결정 사항은 논의 내용과 분리하여 명확하게 나열
- **Slack mrkdwn 문법만 사용**: `*bold*`, `_italic_` 사용 금지. 강조는 backtick(`` ` ``) 코드 포맷만 사용
- `_`가 포함된 이름은 반드시 backtick으로 감싸서 Slack이 italic으로 변환하는 것을 방지
- 섹션 제목: `> ` (blockquote) + backtick으로 키워드 강조
- 항목: `• ` 1단계만 사용. 중첩 불릿(`◦`) 사용 금지 (Slack은 들여쓰기를 무시함)
- 참석자: 이름만 나열 (이메일 제외)
- 액션 아이템: 담당자 + 할 일 + 마감일 형식
- 스레드 본문 5,000자 초과 시 여러 스레드 답글로 분할
- 회의 0건일 때는 이 포맷을 사용하지 않음 (스킬에서 조기 종료)
